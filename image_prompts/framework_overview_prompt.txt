PAPER CONTEXT
-------------
Figure 2 (file: figures/MultiAgentMain.png) for the IEEE SMC paper
"Multi-Agent Retrieval as an Alternative to Classical RAG". This figure
expands the fourth panel of Figure 1 and shows the full pipeline of the
proposed Multi-Agent Retrieval mechanism (internal code name:
`hybrid_fast`). Unlike classical RAG, the retriever is not the top-level
controller; it is a gated tool inside a coordination policy.

PIPELINE FLOW (left to right, with optional vertical lanes)
-----------------------------------------------------------
1) "Student Query" entry block on the far left.
2) A small block labeled "Lightweight Regime Router" receives the query.
   Show that the router computes five heuristic scores -- evidence
   (s_e), coordination (s_c), rubric, planning, tutoring -- and picks
   the mechanism. Annotate "s_e >= 0.35 triggers retrieval".
3) From the router, three parallel specialist blocks run concurrently
   in an upper lane:
   - "Planner" (emits up to 3 scoped search queries)
   - "Diagnoser" (infers learner level, likely misconception)
   - "Rubric Agent" (compiles rubric checklist from prompt cues)
4) A diamond labeled "Retrieval Gate: s_e >= 0.35  or  regime = EG"
   sits to the right of the specialists in the middle lane.
   - DASHED branch -> "Hybrid Retrieval Stack" (word TF-IDF +
     char n-gram + latent SVD -> Reranker -> MMR Packer -> <= 4 chunks).
     Label the gate branch "require_retrieval = True".
   - SOLID branch -> bypass retrieval.
5) The specialist outputs and the (optional) retrieved chunks merge into
   a shared block labeled "AgentContext" (learner state, plan, rubric,
   chunks).
6) "AgentContext" -> "Tutor Agent" block which produces the draft.
7) "Tutor Agent" -> DASHED arrow to "Critic Agent" (optional revision).
8) From "Critic Agent" a DASHED fallback arrow loops back to the
   Retrieval Stack labeled "Fallback (no chunks  &&  s_e >= 0.45)", and
   then back to the Tutor for a single re-run.
9) Final output block on the far right: "Final Tutor Response".
