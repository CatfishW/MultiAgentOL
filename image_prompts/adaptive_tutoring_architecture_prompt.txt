PAPER CONTEXT
-------------
Auxiliary / alternative Figure 1 rendering for the IEEE SMC paper
"Multi-Agent Retrieval as an Alternative to Classical RAG". The goal of
this alternate rendering is to make the CONCEPTUAL CLAIM of the paper
visible at a glance: in classical RAG the retriever is the top-level
controller, while in our multi-agent retrieval mechanism the retriever
is a gated tool inside a coordination policy. This file is kept as an
exchangeable companion to execution_families_prompt.txt; either can
produce figures/family.png.

VISUAL STRUCTURE (two paired panels side by side)
-------------------------------------------------
Left panel: "Classical RAG (retrieval is the controller)".
- A Student Query box at top.
- An arrow down to a large Retriever block that dominates the panel.
- The Retriever sits visually *above* a small Tutor block, conveying
  that the retriever is in charge.
- Label the Retriever "top-level controller: retrieve-then-read".

Right panel: "Multi-Agent Retrieval (our proposal: coordination is the
controller; retrieval is a gated tool)".
- A Student Query box at top.
- An arrow into a central "Coordination Stack" block that contains the
  Planner, Diagnoser, and Rubric agents running in parallel.
- Beneath the Coordination Stack, a diamond labeled
  "Gate  s_e >= 0.35 ?".
- From the gate, a DASHED branch drops to a smaller Retriever block
  (conditional tool), and a SOLID branch bypasses it.
- Both branches converge into the Tutor.
- An optional Critic loops back into the Tutor with a dashed arrow, and
  a second dashed fallback loop goes from the Critic back to the
  Retriever.
- Label the Coordination Stack "top-level controller"; label the
  Retriever "tool (called only when needed)".

CONTRAST MARKERS
----------------
- In the left panel the Retriever box is large and placed on the main
  axis. In the right panel the Retriever box is small and offset.
- Keep the Tutor block the same size in both panels so the reader sees
  that only the controller identity changed, not the downstream reader.
- Below both panels, add a shared caption strip:
  "Same backbone, same corpus, same reranker; only the control
  structure around retrieval differs."

STYLE
-----
Flat vector illustration, modern scientific infographic aesthetic, white
background. Neutral gray and muted slate for the classical RAG panel;
teal / blue-green accent reserved for the Coordination Stack, the Gate,
and the dashed retrieval branch in the Multi-Agent Retrieval panel.
Solid outlines for always-active paths, dashed outlines for conditional
and fallback paths. No 3D, no decorative clutter, no hand-drawn styling.
Labels must stay legible at two-column IEEE size.
